# Chrome 网站收藏管理助手

## 页面线框图说明 + shadcn 组件映射 + 页面组件树

本文档基于“尽量直接使用 `shadcn/ui` 组件”的实现思路整理，目标是：
- 不单独重画一套复杂 UI 体系
- 优先使用现成组件快速搭建
- 在少量业务区域做轻量组合

## 1. 设计原则

### 1.1 总体原则
- 以 `shadcn/ui` 现成组件为主
- 页面结构清晰优先于视觉花样
- 用布局、间距、边框、阴影和状态色来建立层级
- 保留拖拽状态和收藏管理的核心交互

### 1.2 实现原则
- 顶部工具栏、弹窗、菜单、表单、提示优先直接用 `shadcn/ui`
- 左侧分类项和右侧卡片用 `Card`、`Button`、`DropdownMenu`、`Dialog` 组合而成
- 尽量不额外设计复杂插画、图形背景和定制皮肤

### 2.1.2 顶部工具栏

- 最左侧：产品名 `OneMoreMark`

- 中间：搜索框

- `Button` 或纯文本作为 `OneMoreMark` 标识
- `Button` 作为收起左侧分栏按钮
- `Input` 作为搜索框
- `DropdownMenu` 作为右上角统一菜单


- 高度：`64px`

- `OneMoreMark` 固定在最左侧
- 搜索框占据工具栏主要宽度
- 菜单固定在最右侧

#### 右上角菜单项
- 全部合并
- 全部展开
- 导出

## 2.5 插件弹窗页线框图

### 2.5.1 页面结构

```text
+----------------------------------+
```

### 2.5.2 交互说明
- 不展示当前页面信息区域

- 当前页如果已在某分类中，分类项右侧显示“已添加”

### 2.5.3 新增分类交互
- 点击加号
- 弹出小型输入区域或轻量弹窗

- 创建成功后插入到分类列表顶部

### 2.5.4 建议组件组合
- 顶部标题行：`Button + Text`

## 5. 样式落地建议

## 5.1 尽量少定制的做法
- 颜色走 `shadcn` 变量体系
- 圆角走统一 token
- 阴影、边框、背景用 Tailwind 工具类完成
- 不额外重做按钮、输入框、菜单样式

## 5.2 只做轻量定制的区域
- 分类项选中态
- 卡片 hover 态
- 拖拽中态
- 左侧分类接收拖拽时的高亮态
- 网站标题单行 / 双行时的排版切换

## 5.3 拖拽与 shadcn 的组合方式
- `dnd-kit` 负责拖拽行为
- `Card` / 容器布局负责可视化容器
- 通过 `className` 切换拖拽状态样式

推荐状态类思路：
- 默认：普通边框
- hover：`shadow-sm` + `border-accent`
- dragging：`opacity-80 scale-[1.02] shadow-lg`
- drag-over：`ring-2 ring-primary`

## 6. 最终建议

如果你希望尽量快地落地，不想单独做复杂视觉设计，那么“直接用 `shadcn/ui` 组件搭建”是完全可行的。

最推荐的方式不是重新设计一套 UI，而是：
- 用 `shadcn/ui` 作为全部基础组件
- 用 `Card + Button + DropdownMenu + Dialog + ScrollArea` 拼出主界面
- 仅在拖拽和选中状态做少量业务样式增强

这样做的优点：
- 上手快
- 工程简单
- 风格统一
- 后续维护成本低
