# Chrome 网站收藏管理助手

## 技术选型判断与 UI 方案

本文档基于当前 PRD，拆分为：
- 技术方案是否合适
- 功能模块清单
- 页面结构设计
- UI 视觉与交互设计
- 推荐技术实现方案

## 1. 技术选型判断

### 1.1 React 是否合适
合适，且是比较推荐的方案。

原因：
- 新标签页是一个中等复杂度的单页应用，包含分类列表、卡片分组、拖拽、搜索、导入导出、主题切换等，React 很适合管理这类状态驱动界面
- 插件弹窗页也属于典型的小型交互页面，React 可以复用状态、组件和样式体系
- 后续如果扩展设置页、导入页、批量操作页，也能继续沿用同一套组件和工程结构

结论：
- `React` 适合作为核心 UI 框架

### 1.2 shadcn 是否合适
合适，但建议作为“基础组件层”使用，而不是完全照搬默认样式。

原因：
- 你的产品界面不是纯后台表单，更偏“内容管理 + 卡片浏览 + 拖拽整理”
- `shadcn/ui` 在输入框、弹层、下拉菜单、对话框、按钮、滚动区、命令面板这些基础能力上很好用
- 但分类栏、网站卡片、拖拽态、高亮态、空状态这些核心区域，建议做业务化定制，不要直接套默认组件外观

结论：
- `shadcn/ui` 可以用
- 适合作为基础组件库
- 不建议把整个主界面做成“标准后台管理风格”

### 1.3 Base UI 是否合适
如果你说的 `baseui` 是“基础交互组件思路”，那是合适的。
如果你是指单独引入另一套重量级 UI 库，则不建议和 `shadcn/ui` 混用。

原因：
- Chrome 插件项目本身不大，同时使用两套 UI 体系会增加样式冲突和维护成本
- `shadcn/ui + Tailwind + 自定义业务组件` 已足够覆盖本项目

结论：
- 推荐保留一套主 UI 体系
- 更建议：`React + Tailwind CSS + shadcn/ui`

### 1.4 推荐结论
推荐技术栈：
- `React`
- `TypeScript`
- `Tailwind CSS`
- `shadcn/ui`
- `dnd-kit`
- `CRXJS` 或 `Plasmo`

其中更推荐：
- 如果你想要更接近标准前端工程：`Vite + CRXJS`
- 如果你想更快搭建 Chrome 插件：`Plasmo`

对你这个项目，我更推荐：
- `React + TypeScript + Vite + CRXJS + Tailwind + shadcn/ui + dnd-kit`

原因：
- 结构清晰
- 适合新标签页和弹窗双入口
- 方便自己掌控构建与目录结构

## 2. 功能模块清单

### 2.1 数据管理模块
职责：
- 管理分类数据
- 管理收藏卡片数据
- 管理设置数据
- 封装本地存储与同步存储

核心能力：
- 新增 / 编辑 / 删除分类
- 新增 / 编辑 / 删除收藏
- 更新分类顺序
- 更新卡片顺序
- 更新卡片所属分类
- 保存主题模式和侧边栏展开状态

### 2.2 插件弹窗模块
职责：
- 获取当前页面信息
- 快速收藏当前页面
- 快速新建分类

核心能力：
- 读取当前激活标签页
- 展示分类列表
- 将当前页面加入指定分类
- 避免同分类重复收藏

### 2.3 新标签页主页模块
职责：
- 展示完整收藏数据
- 负责主要管理交互

核心能力：
- 左侧分类列表
- 右侧分类分组卡片区
- 搜索
- 主题切换
- 分类栏折叠
- 导入导出入口
- 批量保存标签页入口

### 2.4 拖拽模块
职责：
- 管理分类拖拽
- 管理卡片拖拽
- 管理跨分类拖拽
- 管理拖入左侧分类列表

核心能力：
- 左侧分类排序
- 分类内卡片排序
- 右侧卡片拖入右侧其他分类
- 右侧卡片拖入左侧分类项，自动落到末尾

### 2.5 搜索模块
职责：
- 根据标题和 URL 搜索卡片

核心能力：
- 实时筛选
- 模糊匹配
- 按原分类分组显示结果

### 2.6 导入导出模块
职责：
- 导入 Chrome 书签 HTML 文件
- 导出兼容 Chrome 的书签文件

核心能力：
- 解析书签结构
- 分类映射
- 去重策略
- 导出结构生成

### 2.7 批量标签页归档模块
职责：
- 批量保存当前窗口标签页
- 自动创建临时分类
- 关闭已保存标签页

核心能力：
- 获取当前窗口标签页
- 过滤特殊页面
- 批量创建收藏
- 批量关闭标签页

### 2.8 主题模块
职责：
- 管理浅色 / 深色 / 跟随系统

核心能力：
- 主题切换
- 主题持久化
- 多页面共享主题状态

## 3. 页面结构设计

## 3.1 页面列表
- 插件弹窗页 `popup`
- 新标签页主页 `newtab`
- 可选设置页 `options`

说明：
- 一期即使不单独做设置页，也建议预留入口

## 3.2 插件弹窗页结构

### 目标
让用户在最短路径内完成“当前网页收藏到某分类”。

### 页面区域
- 顶部信息区
- 分类列表区
- 快速新建分类区
- 操作反馈区

### 推荐布局
1. 顶部
展示当前网页 favicon、标题、域名

2. 中部
展示分类列表，列表项可点击收藏

3. 底部
输入框 + 新建分类按钮

### 设计建议
- 弹窗尺寸控制在窄而高的结构，比如 `360 x 520`
- 分类列表优先展示可滚动区域
- 当前网址若已在某分类收藏，可在对应分类右侧显示“已收藏”

## 3.3 新标签页主页结构

### 页面总体布局
- 顶部工具栏
- 左侧分类栏
- 右侧内容区

推荐结构：

1. 顶部工具栏
- 全局搜索框
- 导入按钮
- 导出按钮
- 一键保存全部标签页按钮
- 主题切换按钮

2. 左侧分类栏
- 分类标题
- 新建分类按钮
- 分类列表
- 折叠 / 展开按钮

3. 右侧内容区
- 分类分组区
- 每个分组下展示卡片网格

### 页面响应逻辑
- 桌面宽屏：左侧固定宽度，右侧自适应
- 小屏笔记本：左侧可折叠为窄栏
- 极窄情况下：左侧转为抽屉层，但优先做桌面版即可

## 4. UI 设计方案

## 4.1 整体风格方向
建议不要做成“传统书签管理器”那种密集列表，而是做成：
- 清爽
- 信息层级明确
- 卡片轻量
- 偏效率工具
- 带一点桌面工作台感

推荐气质：
- 类似“资源面板 + 灵感库 + 工作台”的中性风格

### 视觉关键词
- 干净
- 有秩序
- 柔和边界
- 明确拖拽反馈
- 强调可浏览性

## 4.2 布局尺寸建议

### 左侧分类栏
- 展开宽度：`240px`
- 折叠宽度：`72px`

### 顶部工具栏
- 高度：`64px`

### 页面边距
- 外层边距：`20px - 24px`

### 分类分组间距
- 分组上下间距：`24px`

### 卡片网格
- 推荐最小卡片宽度：`220px`
- 网格方式：`repeat(auto-fill, minmax(220px, 1fr))`
- 卡片间距：`16px`

## 4.3 颜色与主题建议

### 浅色模式
- 页面背景：偏暖灰白或冷灰白，不要纯白刺眼
- 左栏背景：略深于页面背景
- 卡片背景：纯色白卡
- 边框：低对比灰色
- 强调色：建议使用蓝绿系或偏青色，而不是通用紫色

建议参考：
- 背景：`#F5F7FB`
- 侧栏：`#EEF2F7`
- 卡片：`#FFFFFF`
- 主文字：`#18212F`
- 次文字：`#5B6472`
- 强调色：`#1D8F6A`

### 深色模式
- 页面背景：深蓝灰，不要纯黑
- 侧栏背景：更沉一点
- 卡片背景：略浮起的深灰蓝
- 边框：低对比度描边
- 强调色：沿用浅色主色，提升一致性

建议参考：
- 背景：`#0F1722`
- 侧栏：`#131D2B`
- 卡片：`#192435`
- 主文字：`#E8EEF8`
- 次文字：`#9AA7BA`
- 强调色：`#3BC792`

## 4.4 字体建议
如果希望更现代一点：
- 中文：`PingFang SC`, `Microsoft YaHei`, `Noto Sans SC`
- 英文和数字：`Inter`, `Geist`, `IBM Plex Sans`

推荐搭配：
- 主界面：`Inter + PingFang SC`

## 4.5 顶部工具栏设计

### 组成
- 左侧：搜索框
- 右侧：导入、导出、一键收纳、主题切换

### 搜索框建议
- 宽度：`320px - 420px`
- 左侧搜索图标
- placeholder：
  `搜索标题或网址`

### 右侧按钮建议
- 导入：次级按钮
- 导出：次级按钮
- 一键保存全部标签页：强调按钮
- 主题切换：图标按钮

### 一键保存按钮建议
文案：
- `保存当前全部标签页`

不要写成：
- `全部关闭`

因为容易让用户紧张。

## 4.6 左侧分类栏设计

### 分类项结构
- 分类图标
- 分类名称
- 数量徽标
- 更多操作按钮

### 分类项状态
- 默认态
- hover 态
- 选中态
- 可接收拖拽态

### 拖拽到左侧分类的视觉反馈
当用户拖动右侧卡片靠近左侧分类项时：
- 分类项整体高亮
- 左侧出现强调色边条
- 分类项背景增强
- 鼠标释放后出现短暂成功反馈

这块很关键，它会直接决定“拖到左边是否自然”。

## 4.7 右侧分类分组设计

### 分组头部
每个分类分组建议包含：
- 分类名
- 数量
- 分组级操作按钮

示例：
- `设计灵感 12`
- `开发工具 8`

### 分组卡片区
- 使用响应式网格
- 支持空分组作为可拖入目标区域

### 空分组提示
如果某个分类没有卡片：
- 展示占位文案
- 仍然允许拖拽卡片进入

推荐文案：
- `拖动卡片到这里`

## 4.8 网站卡片设计

### 卡片内容
- favicon
- 标题
- URL 简写
- 操作按钮

### 卡片推荐结构
1. 顶部
- favicon
- 标题
- 更多操作按钮

2. 底部
- 域名 / URL

### 卡片样式建议
- 高度控制在 `84px - 96px`
- 圆角 `14px - 16px`
- 阴影轻微，不宜太重
- hover 时轻微上浮和描边增强

### 卡片操作
- 点击主体：打开网址
- 更多菜单：
  - 编辑标题
  - 复制链接
  - 删除

### 拖拽态建议
- 被拖动卡片稍微放大 `1.02`
- 阴影增强
- 透明度略降
- 拖拽占位区域清晰

## 4.9 弹窗 UI 设计

### 风格方向
比新标签页更紧凑，但仍保持同一视觉体系。

### 建议结构
- 顶部当前网页卡片
- 中部分类列表
- 底部新建分类输入区

### 当前网页卡片内容
- favicon
- 标题
- URL

### 分类列表项
- 分类名
- 收藏状态

### 收藏反馈
- 点击后弹出 toast：
  - `已收藏到「设计灵感」`

## 4.10 空状态设计

### 首次使用
右侧大空状态卡片：
- 标题：`还没有收藏内容`
- 说明：`浏览网页时点击插件按钮，即可保存到分类中`
- 按钮：`新建第一个分类`

### 搜索无结果
- 标题：`没有找到匹配的网站`
- 说明：`试试搜索更短的关键词或网址片段`

### 分类为空
- 文案：`这个分类还没有内容，拖动卡片到这里`

## 5.2 状态管理建议
推荐优先使用：
- `Zustand`

原因：
- 比 Redux 更轻
- 适合插件和中小型应用
- 拖拽、搜索、主题、分类卡片管理都很好组织

建议 store 划分：
- `useCategoryStore`
- `useBookmarkStore`
- `useSettingsStore`
- `useUIStore`

## 5.3 拖拽方案建议
推荐：
- `dnd-kit`

原因：
- 可控性强
- 适合复杂拖拽场景
- 支持列表排序、网格排序、多容器拖拽
- 更容易实现“右侧卡片拖到左侧分类项”

需要实现的拖拽场景：
- 左侧分类排序
- 右侧分类内卡片排序
- 右侧卡片跨分类拖动
- 右侧卡片拖入左侧分类项并落到末尾

实现要点：
- 分类列表和右侧分类分组都作为 droppable 容器
- 卡片拖入目标分类后统一执行：
  - 更新 `categoryId`
  - 重新计算目标分类下最后顺位
  - 原分类顺序重排

## 5.4 存储方案建议

### 本地存储
- `chrome.storage.local`

用途：
- 存主要数据
- 容量相对更宽松

### 同步存储
- `chrome.storage.sync`

用途：
- 存设置
- 存轻量索引或小体积收藏数据

建议策略：
- 默认数据主写本地
- 再同步一份压缩后的核心数据
- 如果超出 `sync` 限额，则提示用户“当前仅本地保存”

## 5.5 导入导出方案

### 导入
- 通过文件选择器读取 Chrome 书签 HTML
- 前端解析 DOM 结构
- 将文件夹映射为分类
- 将链接映射为卡片

### 导出
- 将当前分类和收藏结构组装成兼容的书签 HTML
- 使用 Blob 下载

## 5.6 新标签页实现方案
在扩展配置中将 `newtab` 作为覆盖页入口。

页面职责：
- 承载主要管理体验
- 展示所有分类和卡片
- 提供完整操作能力

## 5.7 插件弹窗实现方案

### 核心逻辑
- 读取当前激活标签页
- 提取 `title`、`url`
- 优先使用标签页图标作为 favicon
- 展示分类列表
- 点击后完成保存

### 注意点
- 某些页面不可读取或不可保存，需要前置过滤

## 6. 最终建议

对于这个项目，推荐方案是：
- 用 `React` 做页面层
- 用 `Tailwind + shadcn/ui` 做基础 UI
- 用 `dnd-kit` 做所有拖拽交互
- 用 `chrome.storage.local + chrome.storage.sync` 做存储
- 用 `Vite + CRXJS` 组织扩展工程

UI 上建议突出以下体验：
- 左侧分类清楚、可折叠
- 右侧卡片整洁、轻量、好拖动
- 拖到左侧分类时反馈明确
- 不做传统后台风，做成更像个人工作台

如果继续下一步，建议直接做两份输出：
- 一份“页面线框图说明”
- 一份“前端组件拆分清单”
